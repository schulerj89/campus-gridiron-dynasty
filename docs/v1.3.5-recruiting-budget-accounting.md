# v1.3.5 Recruiting Budget Accounting Decision

## Code-Review Expert Finding

`refreshRecruitingBudget` updated `seasonBudget` and `pointsRemaining` after Program Blueprint changes, but left `pointsSpent` untouched. If the user had already spent recruiting points, blueprint changes could leave `pointsRemaining + pointsSpent` above the current season budget.

## Patch Decision

After recalculating the recruiting budget, recompute `pointsSpent` from the current `seasonBudget - pointsRemaining`.

## Why

- Recruiting budget numbers are user-visible and drive recruiting strategy.
- Internal accounting should stay consistent after both manual blueprint allocation and auto-build.
- The change preserves existing point refunds and only fixes the derived accounting total.

## Validation

- Added coverage for spending recruiting points, then applying manual and auto-built Program Blueprint changes.
- Smoke refreshes Recruiting and Program screenshots with the visible patch version.
