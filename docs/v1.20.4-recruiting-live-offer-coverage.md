# v1.20.4 Recruiting Live Offer Coverage

## Expert Finding

The code-review expert found that Recruiting Need Command counted any user offer for a position, including recruits who had already committed to another school or already pledged to the user. That made need coverage look stronger than the live recruiting board actually was.

## Decision

Treat this as a patch-level UI integrity fix. Need Command offer counts should show live actionable offers only; committed prospects are already represented by pledge counts or are no longer recruitable.

## Fix

- Added `liveOfferCountForPosition` in the recruiting service.
- Updated Need Command to use the shared helper.
- Added regression coverage for one open offer, one committed-elsewhere offer, and one user pledge at the same position.
