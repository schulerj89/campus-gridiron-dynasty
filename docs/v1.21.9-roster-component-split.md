# v1.21.9 Roster Component Split

## Release Type

Patch release. This is a small local refactor around roster and depth chart components.

## Code-Review Expert Decision

The code-review/design expert found that `Roster` owned team selection, filtering, roster rows, depth-chart editing, and modal state in one block. That made small UI fixes harder to review and increased the chance of future layout drift.

## Why

Separating the picker, roster list, and depth chart panel keeps each UI area scoped while avoiding a broad file move. The player modal and state ownership stay in place, which keeps the patch low-risk.

## Changes

- Added local `RosterTeamPicker`, `RosterList`, and `DepthChartPanel` components in `src/App.tsx`.
- Kept player modal state and update callbacks owned by `Roster`.
- Preserved smoke test IDs for roster, depth chart, player modal, and other-team roster flows.
