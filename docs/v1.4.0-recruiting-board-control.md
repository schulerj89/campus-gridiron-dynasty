# v1.4.0 Recruiting Board Control Decision

## Problem

The user could add, offer, scout, and pitch prospects, but could not cleanly back out after changing priorities. Auto-recruit also overreacted to a single largest roster gap, which made the board feel like it ignored broader class needs.

## Decision

Add explicit board removal and scholarship rescind actions, and rebuild auto-recruit board selection around position board targets plus roster needs. Removing or rescinding frees board/offer state but keeps spent points as sunk cost; commitment refunds remain the only point return path.

## Why

- Board management needs reversible roster-planning decisions without turning recruiting points into a free trial system.
- A 35-player board should reflect class shape, not just the largest position target.
- Keeping this logic in `src/sim/recruiting.ts` lets the UI, debug tools, and weekly automation share the same behavior.

## Validation

- Added regression coverage for rescind/remove behavior, sunk point accounting, and balanced auto-recruit board position mixes.
- Refreshed recruiting smoke screenshots with the visible `v1.4.0` badge.
