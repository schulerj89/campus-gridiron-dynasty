# v1.3.1 Legacy Save Integrity Decision

## Code-Review Expert Recommendation

Patch legacy save normalization first. Older local saves can be missing fields added by recent releases, including debug flags, debug logs, recruiting action defaults, dynasty history containers, weekly awards, and poll movement arrays.

## Why

- Continuing an older save should not crash the dashboard, rankings page, recruiting page, or weekly advance.
- Local-only storage means compatibility depends on the client-side normalizer, not a server migration.
- Patch-level hardening reduces support risk without changing save semantics for current dynasties.

## Patch Scope

- `normalizeDynastyState` now fills missing debug fields, recruiting defaults, weekly awards, dynasty history, team history, and ranking movement arrays.
- Ranking snapshots with missing `allEntries` still rebuild full 1-70 ranking data while keeping safe movement arrays.
- The same integrity pass also fixed completed-season team history to record only current-season awards, preventing Program Record Book award inflation.

## Validation

- Added migration tests for older save shapes.
- Added a regression test for stale player career awards leaking into season history.
- Full smoke refreshes visible version and Awards/Record Book screenshots.
