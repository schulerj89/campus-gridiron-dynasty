# v1.12.6 Patch Decision - Active Save Recovery

## Decision

When the active localStorage pointer is missing or points at a missing record, scan the local IndexedDB saves and recover the newest dynasty by save timestamp.

## Why This Is a Patch

This is local data recovery hardening for the existing single active-save flow. It does not add multi-save UI or alter dynasty data; it prevents a valid IndexedDB save from becoming unreachable because localStorage drifted.

## Integrity Notes

- A valid active pointer still loads that exact save first.
- Missing or stale active pointers fall back to the newest saved dynasty using `updatedAt`, then `createdAt`.
- Successful fallback rewrites the active pointer and compact summary so future Continue loads are direct.
- Empty IndexedDB stores still return no active save cleanly.
