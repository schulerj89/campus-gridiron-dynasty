# v1.3.1 Award History Dedupe Decision

## Code-Review Finding

Team history was reading `player.awards` when the offseason closed. That field is cumulative across a player's career, so a player who won an award in a prior year could cause the next season's history and Program Record Book totals to count the old award again.

## Patch Decision

Record completed-season history from the current `SeasonAwards` object instead of the roster's cumulative career award arrays.

## Why

- Keeps the Program Record Book honest across a 20-year dynasty.
- Avoids bloating award counts as decorated players remain on the roster.
- Preserves player career award history while separating it from season-history summaries.

## Validation

- Added a regression test that seeds a player with a legacy award and confirms offseason history does not record it when no current season awards exist.
- Smoke coverage refreshes the Awards and Program Record Book screenshots with the patch version.
