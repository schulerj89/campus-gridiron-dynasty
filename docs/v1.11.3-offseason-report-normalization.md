# v1.11.3 Patch: Offseason Report Normalization

## Code-Review Finding

Legacy offseason saves can be missing `topClasses` or per-team report arrays. Current offseason panels expect arrays for class rankings, departures, signees, walk-ons, progressions, and program changes.

## Decision

Harden `normalizeOffseasonReport` to default missing report arrays before the app renders or advances a loaded save.

## Why Patch

This is a saved-data compatibility bug fix. It is low risk because it only fills missing arrays and does not change current save generation or simulation outcomes.

## Verification

- Added a storage regression for older offseason saves missing report arrays.
- Refreshed smoke screenshots, including offseason dashboard coverage.
- Full release validation run for the patch.
