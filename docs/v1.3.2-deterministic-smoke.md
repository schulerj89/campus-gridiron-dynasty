# v1.3.2 Deterministic Smoke Decision

## Code-Review Expert Recommendation

Make smoke runs and screenshots deterministic. The prior smoke config could reuse an existing server on `4173`, and new dynasties were seeded from `Date.now()`, which made tracked screenshots vary between runs.

## Why

- A stale server on the preview port can make smoke pass or fail against the wrong application.
- Tracked screenshot artifacts should reflect code changes, not a different random dynasty world.
- WebKit/mobile coverage is only meaningful when the target app and data seed are controlled.

## Patch Scope

- `scripts/run-smoke.mjs` finds an available local port and passes it to Playwright.
- Playwright preview startup uses `--strictPort` and `reuseExistingServer: false`.
- Smoke specs launch with fixed `?seed=` values.
- The app accepts a positive `seed` query param when starting a new dynasty, falling back to `Date.now()` for normal user play.
- The unit-test script runs sim-heavy files sequentially with a larger per-test timeout so validation is stable on busy local machines.

## Validation

- Run `npm run smoke` and confirm the launch page shows Campus Gridiron Dynasty, not another local app.
- Re-run smoke to verify the screenshot set does not churn from random seed changes.
