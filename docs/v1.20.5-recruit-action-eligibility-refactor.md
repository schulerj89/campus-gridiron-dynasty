# v1.20.5 Recruit Action Eligibility Refactor

## Expert Finding

The code-review expert found that the recruiting card and recruit modal duplicated Add, Offer, Scout, Pitch, and Rescind disabled-state rules. Recent fixes had to be made in both places, increasing the risk that one surface would drift from the other.

## Decision

Treat this as a patch-level refactor because it does not add a new recruiting feature. It consolidates existing UI rules so future recruiting fixes are applied once.

## Fix

- Added a local `recruitActionEligibility` helper in the recruiting UI.
- Routed both `RecruitCard` and `RecruitModal` through the helper for action availability.
- Reused the helper for recruit status and pitch status labels, including clearer committed-player messaging.
