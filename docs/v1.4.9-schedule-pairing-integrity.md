# v1.4.9 Schedule Pairing Integrity

## Decision

Regular-season schedule generation now tracks unordered opponent pairs and prefers unused pairings across the season.

## Why

The weekly scheduler only prevented a team from appearing twice in the same week. It did not remember earlier opponents, so random generation could schedule the same matchup more than once in a regular season.

## Scope

- Added season-level opponent pair tracking to `createSchedule`.
- Replaced the simple greedy weekly matcher with a constrained retrying matcher that prefers teams with fewer unused opponents.
- Added deterministic regression coverage for 12 games per team and no duplicate unordered pairings.

- `npm run test -- src/sim/__tests__/generation.test.ts`
