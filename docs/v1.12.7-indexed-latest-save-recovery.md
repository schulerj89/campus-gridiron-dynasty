# v1.12.7 Patch Decision - Indexed Latest-Save Recovery

## Decision

Add an IndexedDB `updatedAt` index and use a reverse cursor to recover the latest save without loading every full dynasty record.

## Why This Is a Patch

This is a storage performance and retrieval hardening patch. It does not add a new save UI or change dynasty gameplay data; it makes the existing stale-pointer recovery path cheaper and more reliable as saves grow.

## Integrity Notes

- The `dynasties` object store remains keyed by `id`.
- Database version 2 adds the non-unique `updatedAt` index during upgrade.
- Latest-save recovery uses the index when present and keeps the previous full-scan picker as a fallback for compatibility.
