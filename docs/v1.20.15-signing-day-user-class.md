# v1.20.15 Signing Day User Class Default

## Expert Finding

The code-review expert found that the dashboard signing-day panel opened the "All Team Classes" selector on the top-ranked recruiting class, not the user's team. That made the focused offseason dashboard feel detached from the user's own class.

## Decision

Treat this as a patch-level presentation logic fix. Signing results, rankings, and recruit data are unchanged; only the default selected class changes.

## Fix

- Passed the offseason team report ID into the signing-day class panel.
- Defaulted the class selector to that user team when present, with the existing sorted first-class fallback intact.
- Kept manual class selection and pagination behavior unchanged.

## Validation

- Extended the feature smoke flow to assert that the selected signing-day class option contains the active user team name.
- Regenerated `artifacts/screenshots/offseason-all-classes-desktop.png` through `npm run smoke`.
- Ran `npm run build`, `npm run test`, `npm run release:check`, and `npm run smoke`.
