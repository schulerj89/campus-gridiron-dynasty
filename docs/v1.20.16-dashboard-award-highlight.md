# v1.20.16 Dashboard Award Highlight

## Expert Finding

The code-review expert found that the dashboard's Latest National Awards panel rendered the shared `AwardGrid` without `userTeamId`, so user-team highlights appeared on the Awards page but not on the dashboard.

## Decision

Treat this as a patch-level UI integrity improvement. Award calculations and ordering stay unchanged; the dashboard now uses the existing card highlight behavior.

## Fix

- Passed `state.userTeamId` into the overview `AwardGrid`.
- Kept the two-card dashboard limit and empty-state behavior unchanged.
- Made the debug Force User Award command seed a current weekly award card so dashboard QA can verify the highlight deterministically.
- Added smoke coverage that forces a user award and verifies the highlighted card on the dashboard.

## Validation

- Regenerated `artifacts/screenshots/dashboard-awards-highlight-desktop.png`.
- Added unit coverage for the debug weekly award seed.
- Ran `npm run build`, `npm run test`, `npm run release:check`, and `npm run smoke`.
