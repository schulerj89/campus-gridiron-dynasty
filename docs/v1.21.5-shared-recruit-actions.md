# v1.21.5 Shared Recruit Actions

## Release Type

Patch release. This is a small local UI refactor that does not change recruiting mechanics, save data, or visible action availability.

## Code-Review Expert Decision

The code-review expert found that recruit cards and the recruit modal duplicated the Add/Remove, Offer/Rescind, Scout, and Pitch controls. Eligibility was already centralized, but duplicated JSX still increased the chance of label or disabled-state drift.

## Changes

- Added a shared `RecruitActionButtons` component inside `src/App.tsx`.
- Reused it from both `RecruitCard` and `RecruitModal`.
- Preserved compact card labels like `Offer` and full modal labels like `Offer Scholarship`.
- Preserved existing action functions and disabled-state rules.

## Validation

- `npm run build`
- `npm run release:check`
- `npm run smoke`
