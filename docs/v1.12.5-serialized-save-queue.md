# v1.12.5 Patch Decision - Serialized Save Queue

## Decision

Route autosaves and manual saves through a single in-memory queue that writes one dynasty state at a time and skips superseded pending states.

## Why This Is a Patch

This fixes a data-loss race in the existing save behavior. It does not add a new save UI or change the persisted dynasty shape; it only ensures a slower older save cannot overwrite newer progress after rapid state changes.

## Integrity Notes

- The queue serializes writes to the existing IndexedDB save function.
- If multiple states are queued while a save is in flight, only the newest pending state is written after the active save finishes.
- The UI updates save status only for the latest committed save request.
- Manual Save uses the same queue and failure handling as autosave.
