# v1.12.8 Patch Decision - Clear Cancels Pending Saves

## Decision

Invalidate pending and in-flight save queue completions before clearing local dynasty data.

## Why This Is a Patch

This is a storage race fix. It does not change the save schema or add new save UI; it prevents an older queued save completion from updating app state after the user intentionally clears local data.

## Integrity Notes

- `cancelPending` advances the save queue generation and resolves any pending save as skipped.
- In-flight saves that complete after a cancel resolve as stale, so the UI ignores their completion.
- Reset increments the latest UI save request before clearing IndexedDB/localStorage.
- Smoke coverage clears a saved dynasty, reloads, and verifies Continue metadata does not reappear.
