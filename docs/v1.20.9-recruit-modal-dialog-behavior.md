# v1.20.9 Recruit Modal Dialog Behavior

## Expert Finding

The code-review expert found that the regular recruit modal did not match the dialog semantics and backdrop-close behavior already used by signed recruit, player, and game modals.

## Decision

Treat this as a patch-level UI integrity improvement. It aligns existing markup and interaction behavior without changing recruiting mechanics or persisted data.

## Fix

- Added `role="dialog"`, `aria-modal="true"`, and a recruit-specific `aria-label` to `RecruitModal`.
- Added backdrop mouse-down close behavior and stopped propagation for clicks inside the modal.
- Extended the recruiting smoke path to verify dialog attributes and backdrop-close behavior.

- `npm run test`
- `npm run release:check`
- `npm run smoke`
