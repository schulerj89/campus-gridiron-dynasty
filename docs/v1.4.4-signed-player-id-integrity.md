# v1.4.4 Signed Player ID Integrity

## Decision

Signed recruits now receive player IDs that include the dynasty signing year, and offseason signee reports use the same helper.

## Why

Recruit IDs restart each class in active saves, so IDs based only on `recruit.id` and team could collide after several seasons. Duplicate roster player IDs can corrupt depth-chart ordering, modal targeting, box-score attribution, and React rendering keys.

## Scope

- Added `signedPlayerIdForRecruit` in `src/sim/generate.ts`.
- Used the helper for signing-day roster creation and offseason signee reports.
- Added a multi-season regression proving all roster player IDs stay unique.

## Validation

- `npm run test -- src/sim/__tests__/dynasty.test.ts src/sim/__tests__/recruiting.test.ts`
