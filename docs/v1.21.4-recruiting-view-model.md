# v1.21.4 Recruiting View Model

## Release Type

Patch release. This is a small refactor that does not change recruiting mechanics, storage, or visible behavior.

## Code-Review Expert Decision

The code-review expert found that the Recruiting component was doing too much derivation inline: active board filtering, need-command rows, state options, and database filtering/sorting. Keeping those calculations inside the JSX-heavy component made future UI simplification riskier.

## Changes

- Added `src/components/recruitingViewModel.ts`.
- Moved active board, board-full, need command, state option, and filtered recruit list derivation into `buildRecruitingViewModel`.
- Kept pagination, action buttons, modals, and recruit database rendering in the existing component.
- Added unit coverage for board filtering and database filtering/sorting.
