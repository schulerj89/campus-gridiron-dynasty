# v1.21.17 Mobile Usability and Pages Deploy

## Code-Review/Design Expert Decision

The mobile pass should be treated as a patch-level UI and release-infrastructure update. It keeps the existing dynasty features intact, but changes how much each screen exposes at once on iPhone-sized viewports and adds the missing GitHub Pages deployment path.

## Patch Updates

1. Mobile navigation simplification
   - Type: UI refactor.
   - Decision: collapse the dynasty tab rail behind a hamburger section menu on narrow screens.
   - Why: the always-visible tab rail consumed scarce width and encouraged horizontal scanning before the user could act.

2. Mobile pagination density
   - Type: UI improvement.
   - Decision: use smaller mobile page sizes for roster, recruiting board/database, signing classes, preseason development, departures, class rankings, and schedule games.
   - Why: these screens were feature-complete but too long on an iPhone 15 Pro Max viewport.

3. Narrow-screen styling
   - Type: styling bug fix.
   - Decision: tighten mobile stats, recruiting need cards, pagination controls, roster rows, and sticky header sizing.
   - Why: the mobile layout should avoid replacing vertical scrolling with horizontal overflow.

4. GitHub Pages deployment
   - Type: release infrastructure.
   - Decision: add an Actions Pages workflow and a Vite Pages base path for `/campus-gridiron-dynasty/`.
   - Why: the repo did not have a deployment workflow, while `sneak-game` already established the Pages deployment pattern.
