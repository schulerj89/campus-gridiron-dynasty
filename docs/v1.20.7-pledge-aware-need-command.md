# v1.20.7 Pledge-Aware Need Command

## Expert Finding

The code-review expert found that Recruiting Need Command used roster-only position needs while auto recruit already counted user pledges internally. That could leave the UI showing a position as urgent even after the user had enough current pledges to cover the gap.

## Decision

Treat this as a patch-level logic fix. It reconciles existing UI math with existing recruiting behavior and does not change recruiting mechanics or saved data.

## Fix

- Added `userPledgeCountForPosition`.
- Added `positionNeedsWithPledges` as a UI-safe projected need helper while preserving the base roster-only `positionNeeds`.
- Updated Need Command cards and sort-by-need logic to use pledge-aware needs.
- Added regression coverage for a user pledge reducing a QB roster gap.
