# v1.21.29 Mobile Playoff Round Focus

## Code-Review/Design Expert Decision

Treat this as a patch-level mobile UI simplification. The playoff bracket remains complete on desktop, but mobile only shows the round the user is currently advancing through.

## Patch Updates

1. Active round selection
   - Type: UI logic fix.
   - Decision: derive the current playoff round from pending playoff games.
   - Why: Week 13 should show Quarterfinals, Week 14 should show Semifinals, and Week 15/championship recap should show Crown Bowl without extra mobile scrolling.

2. Shared bracket prop
   - Type: small refactor.
   - Decision: add an optional `activeRound` filter to the shared playoff bracket component.
   - Why: Overview can simplify mobile without changing desktop or Awards page behavior.

3. Mobile championship recap
   - Type: mobile UI improvement.
   - Decision: keep the completed championship recap focused on the Crown Bowl bracket column on narrow screens.
   - Why: after the final is played, the phone view should emphasize the title game result instead of replaying the full bracket.

4. Smoke coverage
   - Type: regression coverage.
   - Decision: add iPhone smoke assertions and screenshots for Quarterfinals, Semifinals, and Crown Bowl-only mobile states.
   - Why: this is a stateful flow that can regress when playoff routing or bracket rendering changes.
