# v1.20.17 Signed Recruit Close Button

## Expert Finding

The code-review expert found that the signed-recruit modal close button used `icon-btn`, while the app's standard icon controls use `icon-button`. That made the signing-day modal visually inconsistent with player cards and box score modals.

## Decision

Treat this as a patch-level UI consistency fix. The modal behavior and signed recruit data stay unchanged.

## Fix

- Changed the signed-recruit modal close control to `icon-button`.
- Added smoke coverage that asserts the close button keeps the shared class.
- Refreshed the signing-day recruit modal screenshot.

## Validation

- Regenerated `artifacts/screenshots/signing-day-recruit-modal-desktop.png`.
- Ran `npm run build`, `npm run test`, `npm run release:check`, and `npm run smoke`.
