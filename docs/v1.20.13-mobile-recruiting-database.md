# v1.20.13 Mobile Recruiting Database

## Expert Finding

The code-review expert found that recruiting database rows kept their desktop `970px` minimum width on mobile. That forced horizontal overflow in an otherwise responsive recruiting page.

## Decision

Treat this as a patch-level UI integrity improvement. It is a CSS and smoke-coverage change for an existing view; recruiting state, filters, and mechanics are unchanged.

## Fix

- Added a mobile override for `.recruit-table .table-row`.
- Collapsed recruit database rows into compact two-column cards on smaller screens.
- Allowed long names, hometowns, and statuses to wrap instead of widening the page.
- Added mobile smoke coverage that asserts the recruiting database does not horizontally overflow and captures `mobile-recruiting.png`.

- `npm run test`
- `npm run release:check`
- `npm run smoke`
