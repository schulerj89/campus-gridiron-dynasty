# v1.20.12 Need Command Meter

## Expert Finding

The code-review expert found that Recruiting Need Command progress bars divided coverage by the remaining need value, which could make the meter jump to full too early or use an inconsistent denominator.

## Decision

Treat this as a patch-level UI logic fix. It corrects the visual meter for existing roster, board, and pledge counts without changing recruiting mechanics or saved data.

## Fix

- Added `recruitingNeedCoveragePercent`.
- Calculated open slots from `target - current`.
- Capped covered slots at open slots using active board count plus user pledges.
- Updated Need Command card bars to use the shared helper.
- Added regression coverage for partial, over-covered, and no-open-slot cases.
