# v1.12.2 Patch Decision - Signing-Day Signee ID Integrity

## Decision

Generate offseason report signee `playerId` values from the dynasty class year used by signing day roster creation.

## Why This Is a Patch

This is a data integrity fix for an existing report field. Signing day already creates roster players from committed recruits, but the report was deriving signee IDs from the calendar year, so drill-in data could point at IDs that did not exist on the roster.

## Integrity Notes

- `signRecruitingClass` continues to create incoming freshmen with stable, unique recruit-derived IDs.
- Offseason report enrichment now receives the same dynasty class year and uses it for signee report IDs.
- Regression coverage walks through postseason, offseason recruiting, and signing day, then verifies every report signee ID exists on the matching team roster.
