# v1.20.6 Fully Scouted Recruit Guard

## Expert Finding

The code-review expert found that manual scouting could still spend `SCOUT_COST` on a recruit whose `scoutProgress` was already 100%. Auto recruit avoided this path, but the manual service and button state did not.

## Decision

Treat this as a patch-level bug fix. It prevents duplicate spending on an already completed action without changing recruiting data shape, board limits, or progression rules.

## Fix

- Added a `scoutProgress >= 100` guard to `scoutRecruit`.
- Updated shared recruit action eligibility so fully scouted recruits disable Scout in cards and modals.
- Added a regression test that verifies no point spend, investment mutation, or action log mutation occurs for a fully scouted manual scout.
