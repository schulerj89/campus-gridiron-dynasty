# v1.3.3 Version Hygiene Decision

## Code-Review Expert Recommendation

Add an automated guard for version drift. Patch bumps currently require manual updates to package metadata, lockfile metadata, the visible app version, and the changelog.

## Why

- The app displays its version in both desktop and mobile smoke flows.
- Screenshots and release notes are less useful if they disagree with package metadata.
- A small release check catches drift before commit and push.

## Patch Scope

- Added `scripts/check-release-version.mjs`.
- Added `npm run release:check`.
- The check compares `package.json`, `package-lock.json`, `package-lock` root package metadata, `src/version.ts`, and the latest `CHANGELOG.md` heading.

## Validation

- Run `npm run release:check`.
- Run normal build, test, and smoke validation after each version bump.
