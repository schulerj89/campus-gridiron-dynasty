# v1.3.0 Program Record Book Decision

## Expert Recommendation

Add a program-history surface before expanding another large system. Current official dynasty messaging emphasizes long-running history, weekly storylines, records, and season memory, so the next small feature should make completed seasons matter without increasing weekly input burden.

## Why This Feature

- It gives the user a reason to care about a 20-year save beyond the current roster and poll.
- It reuses existing `Team.history`, `DynastyState.history`, recruiting rank, awards, postseason, and final-rank data instead of adding duplicate save fields.
- It makes prior releases more visible: playoffs, recruiting class rank, player awards, and rankings all feed one readable panel.

## Implementation Notes

- `src/sim/history.ts` derives the record book from completed user-team history.
- The Awards page shows best record, best final rank, best recruiting class, Crown Bowl titles, Summit Four trips, bowl trips, top-10 finishes, individual award count, recent seasons, and award-shelf counts.
- Smoke coverage captures `artifacts/screenshots/program-record-book-desktop.png` after the debug three-season simulation so the screenshot contains real history.

## Guardrails

- Do not store record-book totals as mutable dynasty state.
- Keep the panel readable with empty-state messaging before the first completed season.
- Treat the record book as a summary of completed seasons only; current in-progress season stats belong in the dashboard, rankings, schedule, and stat leaderboard.
