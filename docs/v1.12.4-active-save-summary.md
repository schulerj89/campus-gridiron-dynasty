# v1.12.4 Patch Decision - Durable Active Save Retrieval

## Decision

Resolve save and clear operations only after their IndexedDB transactions complete, then write the active-save pointer and a compact localStorage summary for quick home-screen retrieval.

## Why This Is a Patch

This is durability and retrieval hardening for the existing single active-save flow. It does not change dynasty data or add a new save slot system; it prevents false-positive save status and makes the existing active save faster to identify and continue from the home screen.

## Integrity Notes

- IndexedDB remains the source of truth for the full dynasty save.
- The localStorage summary stores only lightweight metadata: save ID, team name, season position, phase, and update time.
- The active localStorage pointer and summary update only after the IndexedDB write transaction commits.
- If the full IndexedDB save is missing, the summary is cleared instead of offering a stale continue path.
- Save, load, and clear paths close database handles in `finally` blocks.
