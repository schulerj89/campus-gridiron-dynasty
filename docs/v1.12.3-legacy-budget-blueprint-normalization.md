# v1.12.3 Patch Decision - Legacy Budget and Blueprint Normalization

## Decision

Normalize loaded recruiting budgets so spent plus remaining points always equals the season budget, and harden Program Blueprint allocations against non-finite or over-allocated saved values.

## Why This Is a Patch

This is save compatibility hardening. It does not add a new feature; it prevents stale or manually damaged local saves from producing impossible recruiting budgets or `NaN` Blueprint state.

## Integrity Notes

- Saved `pointsSpent` is treated as the source of truth when it is finite, preserving sunk recruiting points.
- If only `pointsRemaining` is usable, spent points are inferred from the season budget.
- Blueprint category allocations normalize to finite integers within category caps and are trimmed to the Blueprint total when a corrupt save is over-allocated.
