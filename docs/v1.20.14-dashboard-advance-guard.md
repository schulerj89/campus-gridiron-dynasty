# v1.20.14 Dashboard Advance Guard

## Expert Finding

The code-review expert found that dashboard-level advance buttons bypassed the header advance flow. Rapid clicks on the dashboard command, postseason bracket, or championship recap could call `advanceWeek` directly before the loading guard took effect.

## Decision

Treat this as a patch-level bug fix because it preserves the existing advance behavior and only routes duplicate dashboard controls through the same guarded action path.

## Fix

- Added an immediate advance lock to the shared advance runner.
- Passed the guarded advance callback, loading state, and current action label into dashboard overview components.
- Updated dashboard, postseason, and championship recap advance buttons to show the same disabled/loading state as the header.

## Validation

- Added `smoke/dashboard-ui.spec.ts` to double-click the dashboard advance button and verify the dynasty lands on Week 2, not Week 3.
- Regenerated `artifacts/screenshots/dashboard-advance-guard-desktop.png`.
- Ran `npm run build`, `npm run test`, `npm run release:check`, and `npm run smoke`.
