# v1.21.3 Recruiting Board Pagination

## Release Type

Patch release. This is a Recruiting page UI polish change that only limits how many active board cards render at once.

## Code-Review Expert Decision

The code-review expert found that an auto-filled recruiting board can create a long wall of cards above the database. The board should remain fully actionable, but it does not need to display every target at once.

## Changes

- Added active Recruiting Board pagination using the existing shared `PaginationControls`.
- Shows eight board targets per page.
- Resets the board page to the first page after a manual Auto Recruit run.
- Keeps board count, board cap, card actions, modals, and recruiting simulation behavior unchanged.

## Validation

- `npm run build`
- `npm run release:check`
- `npm run smoke`
- Added smoke coverage for reaching page 2 of board targets after Auto Recruit.
