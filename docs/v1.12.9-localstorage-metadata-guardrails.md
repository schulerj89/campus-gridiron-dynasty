# v1.12.9 Patch Decision - localStorage Metadata Guardrails

## Decision

Treat localStorage active-save metadata as best-effort and validate that quick summaries match the active save pointer before showing them.

## Why This Is a Patch

This is storage fault containment for quick retrieval metadata. It does not change the IndexedDB save shape or add UI; it prevents blocked, malformed, or mismatched localStorage data from breaking or misleading the existing Continue flow.

## Integrity Notes

- IndexedDB remains the full-save source of truth.
- localStorage get, set, and remove calls are wrapped so browser storage exceptions do not break save/load/clear operations.
- Summary metadata must be structurally valid and match the active pointer ID when a pointer exists.
- Stale summaries are cleared without deleting the full IndexedDB save.
