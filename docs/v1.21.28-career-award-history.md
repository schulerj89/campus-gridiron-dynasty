# v1.21.28 Career Award History

## Code-Review/Design Expert Decision

Treat this as a patch-level award-history integrity fix. It preserves the current-season award display but adds a durable career award trail so player cards and the program record book continue to explain who won what after offseason resets.

## Patch Updates

1. Career player award storage
   - Type: logic fix.
   - Decision: write season award labels into `careerAwards` with the season year while keeping `awards` scoped to the current season.
   - Why: player cards were losing prior-year award context when seasonal awards reset.

2. Named program award shelf
   - Type: display bug fix.
   - Decision: store team history award entries with the winning player name.
   - Why: the program shelf could show the award category but not which user player won it.

3. Repeat-count aggregation
   - Type: small refactor.
   - Decision: aggregate award shelf and player-card award chips by label and show `x2`, `x3`, and higher counts only when repeats exist.
   - Why: repeat winners should be obvious without adding more long scrolling lists.

4. Legacy save normalization
   - Type: save compatibility fix.
   - Decision: default missing `careerAwards` from existing `awards` arrays during load normalization.
   - Why: older local dynasties should open without losing any awards already present on player objects.
